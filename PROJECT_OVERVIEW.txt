🌐 SIMPLIFIED NAMESPACE MODEL (8 PREFIXES)
================================================================================

Prefix       Namespace                                      Purpose
-----------  --------------------------------------------  ---------------------
cf           http://vocab.nerc.ac.uk/standard_name/        Climate properties
geo1         http://www.w3.org/2003/01/geo/wgs84_pos#     Spatial locations
oboe         http://ecoinformatics.org/oboe/.../oboe-core.owl#  Collections
observation  http://climate-kg.org/observation/            Observations & datasets
qudt         http://qudt.org/schema/qudt/                 Quantity values
sosa         http://www.w3.org/ns/sosa/                   Observations & sensors
unit         http://qudt.org/vocab/unit/                  Units
xsd          http://www.w3.org/2001/XMLSchema#            Data types

DESIGN PRINCIPLE: Minimal prefixes for maximum clarity and simplicity

================================================================================
📊 OUTPUT FORMAT
================================================================================

Filename Pattern:
  eobs_climate_kg_{year_start}_{year_end}_{timestamp}.ttl

Example:
  eobs_climate_kg_2020_2024_20260414_163851.ttl

Content Structure:
  1. Prefix declarations (8 namespaces)
  2. ObservationCollections (one per variable)
  3. SOSA Observations (linked via sosa:hasMember)

Typical Size:
  - 1 year (sampled): ~4 MB, ~10,000 observations
  - 5 years (sampled): ~20 MB, ~50,000 observations
  - Full grid: Much larger (depends on spatial sampling)

Example RDF Pattern:

  # ObservationCollection
  observation:eobs_tg_2020_2024 a oboe:ObservationCollection ;
      sosa:hasMember observation:tg_50p00_3p00_20200101T000000,
                     observation:tg_50p00_3p00_20200102T000000,
                     ... .

  # Individual Observation
  observation:tg_50p00_3p00_20200101T000000 a sosa:Observation ;
      sosa:hasFeatureOfInterest geo1:grid_50p00_3p00 ;
      sosa:hasResult [
          a qudt:QuantityValue ;
          qudt:numericValue 1.523000e+01 ;
          qudt:unit unit:DEG_C
      ] ;
      sosa:madeBySensor sosa:tg_sensor ;
      sosa:observedProperty cf:air_temperature ;
      sosa:resultTime "2020-01-01T00:00:00"^^xsd:dateTime .

KEY FEATURES:
  - Simplified SOSA pattern (6 core properties)
  - No dcterms:isPartOf (observations belong to collection via sosa:hasMember)
  - No sosa:phenomenonTime (only resultTime)
  - Sensor URIs use sosa: prefix (not separate sensor: namespace)
  - Location URIs use geo1: prefix (not separate location: namespace)
  - Collections use oboe:ObservationCollection class

🔍 EXAMPLE SPARQL QUERIES
================================================================================

# Count observations per collection
PREFIX oboe: <http://ecoinformatics.org/oboe/oboe.1.2/oboe-core.owl#>
PREFIX sosa: <http://www.w3.org/ns/sosa/>

SELECT ?collection (COUNT(?obs) as ?count) WHERE {
    ?collection a oboe:ObservationCollection ;
                sosa:hasMember ?obs .
}
GROUP BY ?collection

# Find temperature observations above threshold
PREFIX sosa: <http://www.w3.org/ns/sosa/>
PREFIX cf: <http://vocab.nerc.ac.uk/standard_name/>
PREFIX qudt: <http://qudt.org/schema/qudt/>

SELECT ?obs ?temp ?time WHERE {
    ?obs sosa:observedProperty cf:air_temperature ;
         sosa:hasResult/qudt:numericValue ?temp ;
         sosa:resultTime ?time .
    FILTER(?temp > 25.0)
}
ORDER BY DESC(?temp)

# Get all observations for a location
PREFIX sosa: <http://www.w3.org/ns/sosa/>
PREFIX geo1: <http://www.w3.org/2003/01/geo/wgs84_pos#>
PREFIX qudt: <http://qudt.org/schema/qudt/>

SELECT ?obs ?property ?value ?time WHERE {
    ?obs sosa:hasFeatureOfInterest geo1:grid_50p00_3p00 ;
         sosa:observedProperty ?property ;
         sosa:hasResult/qudt:numericValue ?value ;
         sosa:resultTime ?time .
}
ORDER BY ?time

# Find observations in a time range
PREFIX sosa: <http://www.w3.org/ns/sosa/>
PREFIX xsd: <http://www.w3.org/2001/XMLSchema#>

SELECT ?obs ?value ?time WHERE {
    ?obs sosa:hasResult/qudt:numericValue ?value ;
         sosa:resultTime ?time .
    FILTER(?time >= "2020-01-01T00:00:00"^^xsd:dateTime &&
           ?time <= "2020-01-31T00:00:00"^^xsd:dateTime)
}
